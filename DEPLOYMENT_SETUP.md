# Deployment Setup

## Hertzner
I leased a cheap server at hertzner with ubuntu base 4GB RAM (+/- 6 euro p month)
I manually installed docker and caprover on the server and configured the DNS to connect to my own domain (blinddatingapp.finkingma.com).

## CapRover
https://captain.blinddatingapp.finkingma.com
CapRover is a lightweight deployment manager.

I configured the blinddatingapp in CapRover with services similar to those found in the root docker-compose.yaml.
Using a key that was generated by CapRover, we can push new deployments to CapRover with an image tag (right now we only use 'latest')

## Pipeline
With Github Actions a pipeline gets started per service when a change has been made to the service on the main branch.
This pipeline builds and publishes a new version of the latest docker image of the service.
Finally the latest docker image is pushed to CapRover, triggering an update on http://frontend.blinddatingapp.finkingma.com/
